# PortOne Blesta Gateway

PortOne (formerly iamport) credit card payment gateway integration for Blesta with full support for recurring billing and automated payments.

## Overview

The PortOne Blesta Gateway enables seamless payment processing through PortOne's V1 API, providing a complete solution for subscription-based businesses using Blesta. Built with KG이니시스 (KG Inicis) as the default payment gateway, this module supports any PG provider offered by PortOne through simple configuration changes.

The gateway uses PortOne's JS SDK payment window for secure card registration and issues billing keys that enable automatic recurring payments without requiring customer interaction for each transaction.

## Payment Flow

The gateway follows a secure two-step process:

1. **Card Registration**: Customer registers their card through PortOne JS SDK popup (with 0 amount)
2. **Billing Key Issuance**: PortOne generates a `customer_uid` (billing key)
3. **Card Storage**: Blesta stores the card information securely
4. **Automatic Payments**: Future charges use the billing key via `/subscribe/payments/again` endpoint

## Supported PG Providers

The module supports all PortOne V1 payment gateways that offer billing key functionality:

- **KG이니시스** (html5_inicis) - Fully tested
- 나이스정보통신 (NICE)
- NHN KCP
- 다날 (Danal)
- KICC
- KSNET
- 스마트로 (Smartro)
- 웰컴페이먼츠 (Welcome Payments)
- 키움페이 (Kiwoompay)
- 헥토파이낸셜 (Hecto Financial)

<Note>
Each PG provider has different billing key issuance methods and test MIDs. Refer to PortOne documentation for provider-specific requirements.
</Note>

## Technical Specifications

- **API Version**: PortOne V1 (`https://api.iamport.kr`)
- **Payment SDK**: PortOne JS SDK (`iamport.payment-1.2.0.js`)
- **Default PG**: KG이니시스 (`html5_inicis`)
- **Currency**: KRW (Korean Won)
- **Version**: 1.1.0
