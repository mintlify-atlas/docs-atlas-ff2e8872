## Overview

The PortOne gateway processes one-time payments using the `processCc()` method. This flow uses a billing key (customer_uid) issued by the PortOne JS SDK popup to charge the customer's card.

## Payment Flow

1. Customer opens the PortOne JS SDK popup to register their card
2. PortOne issues a billing key (`customer_uid`) with `bl_` prefix
3. The billing key is passed to `processCc()` via the `reference_id` parameter
4. The gateway charges the card using the `/subscribe/payments/again` API endpoint
5. Payment is verified and the amount is validated

## The processCc() Method

```php
public function processCc(array $card_info, $amount, ?array $invoice_amounts = null)
{
    if (!$this->validateAmount($amount)) {
        return $this->buildPaymentError();
    }

    $requested_amount = (int) round((float) $amount);

    // Billing key (customer_uid) issued via JS SDK popup.
    $customer_uid = $card_info['reference_id'] ?? $_POST['reference_id'] ?? null;

    if (!$this->validateCustomerUid($customer_uid)) {
        $this->Input->setErrors([
            'payment' => [
                'failed' => Language::_('Portone.!error.payment.failed', true),
            ],
        ]);
        return $this->buildPaymentError();
    }

    $merchant_uid = 'bl_' . bin2hex(random_bytes(12));

    $params = [
        'customer_uid' => $customer_uid,
        'merchant_uid' => $merchant_uid,
        'amount' => $requested_amount,
        'name' => 'Blesta Payment',
    ];

    if ($this->currency) {
        $params['currency'] = $this->currency;
    }

    if (!empty($this->meta['channel_key'])) {
        $params['channel_key'] = $this->meta['channel_key'];
    }

    $response = $this->api->payAgain($params);

    if ($response === false) {
        $this->logApiError($this->api->getLastError());
        $this->Input->setErrors([
            'payment' => [
                'failed' => Language::_('Portone.!error.payment.failed', true),
            ],
        ]);
        return $this->buildPaymentError();
    }

    $payment = $response['response'];

    // Verify amount matches request
    if (isset($payment['amount']) && (int) $payment['amount'] !== $requested_amount) {
        $this->api->cancelPayment(['imp_uid' => $payment['imp_uid'], 'reason' => 'Amount mismatch']);
        $this->Input->setErrors([
            'payment' => ['amount_mismatch' => Language::_('Portone.!error.payment.amount_mismatch', true)],
        ]);
        return $this->buildPaymentError();
    }

    $status = $this->mapPaymentStatus($payment['status'] ?? '');

    $this->writeLog('INFO', sprintf(
        'processCc OK | merchant_uid=%s | imp_uid=%s | amount=%d | status=%s',
        $payment['merchant_uid'] ?? $merchant_uid,
        $payment['imp_uid'] ?? '',
        $requested_amount,
        $status
    ));

    return [
        'status' => $status,
        'reference_id' => $payment['merchant_uid'] ?? $merchant_uid,
        'transaction_id' => $payment['imp_uid'] ?? null,
    ];
}
```

## Parameters

### card_info

Array containing card details from the payment form:

- `reference_id` - The billing key (customer_uid) issued by the JS SDK popup
- This parameter is validated using `validateCustomerUid()` to ensure it has the `bl_` prefix

### amount

Payment amount to charge:

- Must be a positive numeric value
- Automatically rounded to the nearest integer (Korean won has no decimal places)
- Validated using `validateAmount()` before processing

### invoice_amounts

Optional array containing invoice breakdown (not used by this gateway).

## JS SDK Popup

The billing key is issued via the PortOne JS SDK popup. When the customer clicks the payment button, the custom card form triggers:

```javascript
// From cc_form.pdt view
IMP.request_pay({
    pg: 'html5_inicis',
    pay_method: 'card',
    merchant_uid: 'bl_' + Date.now(),
    name: 'Card Registration',
    amount: 0,
    customer_uid: 'bl_' + generateUniqueId(),
    buyer_email: customer.email,
    buyer_name: customer.name
}, function(rsp) {
    if (rsp.success) {
        // customer_uid is now registered as a billing key
        document.getElementById('reference_id').value = rsp.customer_uid;
        document.forms[0].submit();
    } else {
        alert('Card registration failed: ' + rsp.error_msg);
    }
});
```

## Amount Verification

The gateway automatically verifies that the charged amount matches the requested amount:

```php
if (isset($payment['amount']) && (int) $payment['amount'] !== $requested_amount) {

    $this->Input->setErrors([
        'payment' => ['amount_mismatch' => Language::_('Portone.!error.payment.amount_mismatch', true)],
    ]);
    return $this->buildPaymentError();

```

If there's a mismatch, the payment is automatically cancelled and an error is returned.

## Payment Status Mapping

PortOne payment statuses are mapped to Blesta statuses:

- `paid` → `approved`
- `ready` → `pending`
- `cancelled` → `void`
- `failed` → `declined`
- anything else → `error`

**Location:** portone.php:886-900

## Return Value

The method returns an array with:

- `status` - Payment status (approved, declined, error, pending)
- `reference_id` - Merchant UID (generated with `bl_` prefix)
- `transaction_id` - PortOne transaction ID (imp_uid)

## Error Handling

Errors are logged and user-friendly messages are displayed:

```php
if ($response === false) {
    $this->logApiError($this->api->getLastError());
    $this->Input->setErrors([
        'payment' => [
            'failed' => Language::_('Portone.!error.payment.failed', true),
        ],
    ]);
    return $this->buildPaymentError();
}
```

Raw PG errors are sanitized and logged to prevent exposing internal codes or test-mode warnings to end users.
